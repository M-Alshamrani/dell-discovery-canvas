# Tutorials

**Audience**: someone new to the project who wants to *learn by doing*.
**Mode**: step-by-step, opinionated, complete-by-the-end-of-the-section.

A tutorial is **not** a reference. It picks one happy path and walks it. If you're tempted to add "alternatively, you could…" — that belongs in a how-to or reference, not here.

## Available tutorials

- **[Run your first workshop](run-first-workshop.md)** — install Docker, start the container, walk through Tabs 1-5 with the Acme FSI demo, generate a Roadmap. ~20 min.
